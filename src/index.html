Software Architecture

  • Use ES6 Modules and Webpack to create a OOP based todo App
  • ‘Todos’ are going to be objects that will be dynamically created, which means either using factories or constructors/classes to generate them.
  • Title, description, dueDate and priority. Also to include notes or even a checklist.
  • Your todo list should have projects or separate lists of todos. When a user first opens the app, there should be some sort of ‘default’ project to which all of their todos are put. Users should be able to create new projects and choose which project their todos go into.
  • Separate application logic (i.e. creating new todos, setting todos as complete, changing todo priority etc.) from the DOM-related stuff, so keep all of those things in separate modules.
  • View all todos in each project (probably just the title and duedate… perhaps changing color for different priorities). Expand a single todo to see/edit its details.
  • Functionality to Delete a todo.
  • Use date-fns which gives a bunch of handy functions for formatting and manipulating dates and times.
  • Using Web Storage API localStorageStorage persistence to save/retrieve data from a JSON file
  • Set up a function that saves the projects (and todos) to localStorage every time a new project (or todo) is created, and another function that looks for that data in localStorage when your app is first loaded.
  • Initially inspect data saved in localStorage using DevTools! i.e. open the Application tab in DevTools and click on the Local Storage tab under Storage. Every time you add, update and delete data from localStorage in your app, those changes will be reflected in DevTools.
  • Since localStorage uses JSON to send and store data, and when you retrieve the data, it will also be in JSON format. Keeping in mind that you cannot store functions in JSON, so find a way to add methods back to your object properties once you fetch them.
